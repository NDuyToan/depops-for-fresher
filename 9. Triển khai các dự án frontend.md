# 9. Triển khai các dự án Frontend

## Lệnh scp — Secure Copy

- Lệnh này cho phép bạn sao chép (copy) file hoặc thư mục giữa các máy tính thông qua SSH (Secure Shell) — tức là toàn bộ dữ liệu được mã hóa, an toàn hơn so với ftp hoặc rcp.
- Cú pháp cơ bản: `scp [tùy chọn] nguồn đích`
- Sao chép toàn bộ thư mục (dùng tùy chọn `-r`)
- Ví dụ: `scp file.txt user@remote_host:/path/to/destination/`

## Unzip 1 file

- cài tool unzip: `apt install unzip`
- chạy: `unzip <tên file> -d <tên folder bug file>`
- `-d` → chỉ định thư mục đích

## Thực hiện

### ta có 1 dự án là todolist

1. ở trong home của user name, tạo 1 folder **project**, rồi tạo từng folder riêng cho từng dự án
2. tạo user cho dự án đó, tên user trùng với tên dự án luôn.

- `adduser todolist`
- cài nodejs, npm, chạy lệnh `npm run build`, `npm run serve` để chạy dự án lên xem.
- cài ` sudo apt install nginx -y`
- tất cả file cấu hình của nginx đều lưu ở `/etc/nginx/`
- `cd` vào `etc/nginx`.
- file config mặc định của nginx nằm ở `site-available/default`, nếu có thay đổi config thì chạy lại lệnh `nginx -t` để kiểm tra xem cú pháp, file có ok không. để áp dụng cấu hình mới thì chạy lệnh `system ctl restart nginx`
- tạo file config cho dự án todolist, `vi conf.d/todolist.conf`

```
server {
    listen 8081;
    root /projects/todolist/dist/;
    index index.html;
    try_files $uri $uri /index.html;
}
```

- lưu file, chạy `nginx -t`, sau đó restart lại.
- test thử ở link :8081, thì vẫn chưa được, xuất hiện lỗi 500 Internal Server Error, bởi vì lúc này todolist set quyền cho user todolist, các user khác thì không có quyền, lúc này cần check xem nginx đang chạy với user nào. vào `vi /etc/nginx/nginx.conf`, user đang chạy là `www-data`, để nginx chạy được thì ít nhất nó phải thuộc vào group `todolist`, chạy lệnh `usermod -aG todolist www-data` để thêm www-data vào todolist
- nếu có nhiều dự án đều dùng nginx, thì nên chạy reload thôi là được, k cần restart
